### 交互原则
1. **上传即分析**：用户上传照片后，自动触发空间分析，无需额外点击
2. **分析即数据**：分析结果持久化到数据库，刷新页面不丢失
3. **一键直达**：常用操作（换风格、调色调）不超过2次点击
4. **实时预览**：调整参数时实时显示效果预览（或快速生成缩略图）

## 模块一：空间智能分析（Space Analyzer）

### 1.1 功能概述
用户上传室内照片后，系统自动分析并输出：
- 空间类型与风格
- 软装家具清单（带位置标注）
- 色彩面板
- 结构线框图

### 1.2 用户流程

```
[点击上传照片] → [上传中...] → [自动分析中...] → [分析完成，展示结果]
                                      │
                                      ▼
                    ┌─────────────────┼─────────────────┐
                    ▼                 ▼                 ▼
              [风格识别]         [家具拆解]          [色彩分析]
                    │                 │                 │
                    ▼                 ▼                 ▼
              显示风格标签       标注家具位置        显示色板
              置信度进度条       可展开详情          配色方案建议
```

### 1.4 技术规格

#### AI 调用

```typescript
Body: {
  imageUrl: string;        // Supabase 存储的图片URL
  projectId: string;       // 项目ID

/*
```

#### 数据库表

```sql
-- 空间分析结果表
CREATE TABLE space_analyses (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  project_id UUID REFERENCES projects(id) ON DELETE CASCADE,
  image_url TEXT NOT NULL,
  
  -- 分析结果（JSONB存储完整结果）
  analysis_result JSONB NOT NULL,
  
  -- 关键字段冗余（方便查询）
  room_type TEXT,
  primary_style TEXT,
  style_confidence NUMERIC,
  
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 家具清单表（从analysis_result解析，独立存储便于查询）
CREATE TABLE furniture_items (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  analysis_id UUID REFERENCES space_analyses(id) ON DELETE CASCADE,
  category TEXT NOT NULL,        -- 沙发、椅子、灯具等
  sub_category TEXT,             -- 三人沙发、落地灯等
  color TEXT,
  color_hex TEXT,
  material TEXT,
  position TEXT,
  estimated_size TEXT,
  bounding_box JSONB,            -- {x, y, width, height}
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

### 2.2 家具替换（虚拟 Staging）

#### 功能描述
用户选择图中某件家具，指定替换为另一件家具（从素材库选或上传参考图），AI 自动完成局部替换，保持透视和光影一致。

#### 界面设计

```

┌────────────────────────────────┐
│                                │
│   ┌────────────────────────┐   │
```

## 数据流总览

```
用户操作                        前端                          后端/AI
─────────────────────────────────────────────────────────────────────────
上传照片                    →  上传至 Supabase Storage    →  返回 URL
                           →  调用 /api/analyze-space    →  多模态 LLM 分析
                           →  展示分析结果面板            →  存储分析结果

点击"一键换风格"            →  调用 /api/generate-style   →  AI 图像生成
                           →  展示生成中状态              →  返回生成图片
                           →  更新方案历史                →  存储方案

点击家具"替换"              →  进入替换流程               →  
选择新家具/上传参考图       →  调用 /api/replace-furniture→  AI Inpainting
                           →  展示结果                   →  返回替换后图片

点击"产品推荐"              →  调用 /api/recommend-products→  向量检索匹配
                           →  展示推荐列表                →  返回相似产品

点击"导出清单"              →  调用 /api/export-list      →  生成 Excel/PDF
                           →  下载文件                   →
```
