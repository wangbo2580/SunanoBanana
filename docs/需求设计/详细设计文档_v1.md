## 1. 产品概述

### 1.1 产品定位

Banana Pro 是一款面向软装设计师的 AI 设计解析工具。核心体验：**上传一张图 → 选择场景 → AI生成 → 获取专业设计图纸**。

### 1.2 MVP功能范围（Phase 1）

| 优先级 | 场景名称 | 入口图标 | 一句话描述 |
|-------|---------|---------|-----------|
| P0 | 🎨 渲染增强 | Palette | SU截图/草图 → 写实摄影级效果图 |
| P0 | 📐 彩平图生成 | Grid3x3 | 平面户型图 → 多风格彩色平面图 |
| P0 | 🎨 颜色搭配分析 | Pipette | 室内照片 → 专业配色分析展板 |
| P0 | 🧩 软装拆解 | LayoutGrid | 室内照片 → 单品家具拼贴展示 |
| P0 | ✏️ 自由修图 | Wand2 | 保留现有图像编辑生成能力 |

### 1.3 设计原则

1. **单图单任务**：每个场景只上传1张主图（特殊场景如"智能布局"可支持多图，Phase 1暂不做）
2. **零配置优先**：上传后点击生成即可，高级选项默认折叠
3. **结果即下载**：生成结果直接展示，支持单张下载
4. **提示词内置**：用户无需写提示词，系统自动根据场景和选项拼接

### 3.2 场景卡片组件设计

**卡片结构**：
- 顶部：图标（48px）+ 场景名称
- 中间：缩略图预览（场景效果示例）
- 底部：一句话描述 + "开始使用 →" 箭头

**交互**：
- hover：卡片上浮 + 阴影加深 + 图标放大
- click：路由跳转到对应场景页面

**视觉规范**：
- 网格布局：桌面端 4列，平板 2列，手机 1列
- 卡片圆角：16px
- 卡片间距：24px
- 背景：轻微渐变或纯色

### 4.1 通用页面组件

```typescript
// 场景页面通用布局
interface ScenePageLayoutProps {
  title: string                    // 页面标题
  description: string              // 页面描述
  children: {                      // 插槽
    upload: ReactNode              // 上传区域（各场景定制）
    config?: ReactNode             // 配置区域（可选）
    result: ReactNode              // 结果展示区域
  }
}

// 上传区通用组件
interface UploadZoneProps {
  accept?: string                  // 接受文件类型
  maxSize?: number                 // 最大文件大小（MB）
  value: string | null             // 已上传图片URL
  onChange: (url: string | null) => void
  label: string                    // 标签文字
  hint?: string                    // 提示文字
}

// 结果展示区通用组件
interface ResultDisplayProps {
  status: 'idle' | 'loading' | 'success' | 'error'
  originalImage?: string           // 原图（用于对比）
  resultImage?: string             // 生成结果
  resultImages?: string[]          // 多图结果
  error?: string
  onDownload?: () => void
  onRetry?: () => void
}
```

### 4.2 生成流程状态机

```
[idle] --上传图片--> [ready]
[ready] --点击生成--> [loading]
[loading] --成功--> [success]
[loading] --失败--> [error]
[success] --重新上传--> [ready]
[error] --重试--> [loading]
[error] --重新上传--> [ready]
```

## 5. 各场景详细设计

### 5.1 渲染增强 (/render)

**场景描述**：将SU模型截图、草图或低质量效果图转换为写实摄影级室内效果图。

**输入**：
- 1张主图（SU截图/效果图/草图）

**配置选项**（默认折叠，点击展开）：

| 配置项 | 类型 | 默认值 | 说明 |
|-------|------|-------|------|
| 风格倾向 | select | "自然写实" | 选项：自然写实/暖调温馨/冷调现代/奢华高级 |
| 添加人物 | switch | false | 是否在场景中添加人物 |
| 人物效果 | select | "虚焦" | 选项：虚焦/清晰（仅开启人物时显示）|
| 输出质量 | select | "4K" | 选项：2K/4K/8K |

**生成逻辑**：
1. 用户上传图片
2. 系统拼接提示词（见第6章）
3. 调用 `/api/generate` 接口
4. 展示生成结果

**结果展示**：
- Before/After 对比滑块（可选）
- 生成结果大图
- 下载按钮

---

### 5.2 彩平图生成 (/floorplan)

**场景描述**：将黑白/线稿平面户型图转换为彩色平面图，支持多种风格。

**输入**：
- 1张平面户型图（线稿/黑白/简单填充）

**配置选项**：

| 配置项 | 类型 | 默认值 | 说明 |
|-------|------|-------|------|
| 风格模板 | select | "手绘质感" | 选项见下方风格列表 |
| 标注语言 | select | "中英双语" | 选项：中文/英文/中英双语 |

**风格模板列表**：

| 风格ID | 风格名称 | 适用场景 |
|-------|---------|---------|
| hand-drawn | 手绘质感 | 方案汇报、概念阶段 |
| marker | 马克笔风格 | 快速表达、草图阶段 |
| mid-century | 中古风 | 复古项目、特色住宅 |
| neo-chinese | 新中式禅意 | 中式项目、文化空间 |
| cartoon | 治愈系卡通 | 儿童空间、展示方案 |
| cream | 奶油风 | 现代住宅、温馨空间 |
| marble-gray | 深灰大理石 | 高端商业、现代奢华 |
| flat-minimal | 2D扁平极简 | 极简项目、快速表达 |

**生成逻辑**：
1. 用户上传平面图
2. 选择风格模板
3. 系统拼接提示词
4. 调用 `/api/generate` 接口
5. 展示生成结果

**结果展示**：
- 原图与彩平图并列对比
- 支持下载

### 6.3 颜色搭配分析

```
请基于上传的室内空间图片，生成一张「颜色搭配分析图」。

1. 颜色模块分析：明确区分主色调、辅色调、点缀色。每个颜色以「色块+HEX色值」形式呈现。每个色块下方标注对应的空间元素。
2. 色彩平衡与点缀：使用圆环图或比例图展示整体配色结构，清楚体现中性色为背景，彩色作为视觉焦点的逻辑。
3. 局部色彩体验：提取3-4个局部空间细节图，每个局部配一句简短关键词说明。
4. 色彩心理氛围：用一段简短中英双语文字总结空间气质，说明颜色如何影响情绪、空间感与居住体验。

整体视觉：单张完整分析展板，建筑设计分析图风格，极简、理性、白色背景。
比例精准，排版专业，信息层级清晰。
```

## 10. 数据流设计

### 10.1 单场景数据流

```
用户操作                    前端状态                     API调用
─────────                  ──────────                   ───────
点击场景卡片        ──→    路由跳转                      ─
上传图片            ──→    调用uploadImage ──→          POST 上传到Supabase
                                                    ──→  返回图片URL
                                                     │
                                                     ↓
选择配置            ──→    更新本地状态                  ─
点击生成            ──→    isLoading = true              ─
拼接提示词          ──→    prompt = buildPrompt(scene, config)
                                                     │
                                                     ↓
调用生成            ──→    ─                         ──→  POST /api/generate
                                                    ──→  AI生成图片
                                                     │
                                                     ↓
展示结果            ──←    isLoading = false           ←── 返回图片URL
下载图片            ──→    浏览器下载                    ─
```

### 10.2 提示词拼接逻辑

```typescript
function buildPrompt(scene: string, config: Record<string, any>): string {
  switch (scene) {
    case 'render':
      return buildRenderPrompt(config)
    case 'floorplan':
      return buildFloorplanPrompt(config)
    case 'color-analysis':
      return buildColorAnalysisPrompt(config)
    case 'furniture-breakdown':
      return buildBreakdownPrompt(config)
    default:
      return config.customPrompt || ''
  }
}

// 示例：渲染增强提示词拼接
function buildRenderPrompt(config: RenderConfig): string {
  const base = PROMPT_TEMPLATES.render.base
  const style = PROMPT_TEMPLATES.render.styles[config.style] || ''
  const figure = config.addFigure 
    ? `人物：室内添加1个人物，${config.figureEffect === 'blur' ? '长曝光形成虚焦' : '清晰呈现'}，人物自然融入场景，不遮挡主要家具。`
    : ''
  const quality = PROMPT_TEMPLATES.render.quality[config.quality] || ''
  
  return [base, style, figure, quality].filter(Boolean).join('\n\n')
}
```
