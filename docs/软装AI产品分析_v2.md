# 软装设计师 AI 工具 —— 产品重构分析

> 目标用户：软装设计师 / 软装助理 / 室内设计师  
> 产品定位：从"通用图片编辑器"升级为"软装设计师的专业 AI 工作流工具"

---

## 一、软装设计师的真实工作流程

通过行业调研（Boss直聘、酷家乐、美间等平台的岗位描述），软装设计师的日常工作流如下：

```
1. 需求沟通期
   → 了解客户风格偏好、预算、空间尺寸
   → 收集参考图、Pinterest/Houzz 灵感

2. 方案构思期
   → 做 Moodboard（情绪板）确定整体风格调性
   → 根据硬装条件（墙面/地面/吊顶已定）搭配软装
   → 选品：家具、灯具、窗帘、地毯、挂画、绿植、艺术品

3. 方案呈现期
   → 将家具单品P到空间照片里（虚拟 staging）
   → 多风格对比方案（现代/法式/侘寂/中古等）
   → 输出效果图、产品清单、报价单

4. 落地执行期
   → 按清单采购、跟踪物流
   → 现场摆场指导
   → 最终效果拍摄存档
```

### 核心痛点

| 痛点 | 具体表现 |
|------|---------|
| **找图慢** | 全网翻单品图，适配空间风格难 |
| **P图累** | 用PS把家具P进空间，透视、光影、比例都要手动调 |
| **风格试错成本高** | 客户说"看看别的风格"，就得重做一套方案 |
| **落地偏差** | 效果图很美，但实际买不到同款家具 |
| **清单难做** | 做完效果图还要手动整理产品清单、尺寸、报价 |
| **沟通低效** | 客户看不懂平面图，需要直观的效果对比 |

## 二、竞品分析（现有 AI 软装工具）

### 国内工具
- **酷家乐**：3D建模+渲染，素材库庞大，但学习成本高，不够轻量
- **美间**：专注软装搭配方案，但AI能力弱，主要靠人工拖拽
- **三维家**：全屋定制为主，软装是附加功能
- **躺平设计家**：新手友好，但专业度不够

### 海外工具
- **Interior AI**：上传照片生成多风格方案，但无法识别具体家具
- **RoomGPT**：拍照→生成主题效果，操作简单但不可控
- **Paintit AI**：聊天式AI + 真实家具推荐（亮点）
- **MyRoomDesigner AI**：效果图 + Moodboard + 购物清单（最接近理想形态）
- **Collov.ai**：虚拟 staging，但家具摆放不够智能
- **ArchiVinci**：渲染质量高，但偏建筑/硬装

### 市场空白（我们的机会）

> **没有一个工具，能同时做到：看懂空间 → 识别软装 → 替换风格 → 推荐可购买产品 → 输出清单**

这正是我们要切入的赛道：
- 比 Interior AI 更专业（懂软装单品）
- 比酷家乐更轻量（无需建模）
- 比 Paintit AI 更垂直（专注软装全工作流）

## 四、核心功能模块详细设计

### 🔍 模块一：空间智能分析（Space Analyzer）

**目标**：上传一张室内照片，AI 自动"看懂"这个空间

#### 1.1 风格识别
- 识别当前空间的装修风格（现代简约/法式/侘寂/中古风/北欧/新中式等）
- 输出风格置信度 + 风格关键词
- 提供风格知识卡片（该风格特点、代表元素、色彩搭配原则）

#### 1.2 软装家具拆解
- 用边界框+标签识别图中所有软装单品：
  - 沙发、茶几、边桌、餐椅、床、床头柜
  - 灯具（吊灯/落地灯/台灯）
  - 窗帘、地毯、抱枕
  - 挂画、绿植、装饰品
- 生成"软装结构图"（只保留家具轮廓和位置，去掉材质纹理）
- 输出每个单品的颜色、材质推测

#### 1.3 色彩分析
- 提取主色调、辅助色、点缀色
- 生成色板（Color Palette）
- 分析配色方案（互补色/类似色/三色搭配等）
- 给出色彩调整建议

#### 1.4 空间结构提取
- 识别墙面、地面、天花板
- 识别门窗位置
- 识别空间类型（客厅/卧室/餐厅/书房）
- 输出简化线框图（Wireframe）

**技术实现**：
- 用多模态大模型（GPT-4o / Claude 3.5 Sonnet / Gemini）做视觉分析
- 返回结构化 JSON，前端渲染标注

---

### 🎨 模块二：智能方案设计（Design Studio）

**目标**：在分析结果基础上，快速生成多种软装方案

#### 2.1 风格迁移 / 一键换风格
- 基于原图结构，生成 N 种不同风格的方案
- 保持硬装不变（墙/地/顶），只改软装
- 支持 10+ 预设风格 + 自定义风格描述
- 输出风格对比画廊（Before/After 并列展示）

#### 2.2 色调调整
- 整体色温调整（暖调→冷调）
- 饱和度调整（素雅→浓郁）
- 明度调整（明亮→沉稳）
- 局部调色（只改沙发颜色、只改墙面颜色等）

#### 2.3 虚拟 Staging（家具替换）
- 选择要替换的家具 → AI 自动遮罩 → 替换为新单品
- 保持透视、光影、比例一致
- 支持从素材库选择，或上传参考图
- 类似现在的 inpaint，但针对家具做了优化

#### 2.4 单品增减
- "去掉地毯" → AI 自动去除并补全地面
- "在沙发旁边加一盏落地灯" → 自动找位置、加物体、调光影
- 支持点击位置 + 自然语言描述

#### 2.5 光影渲染增强
- 调整灯光氛围（自然光/暖光/冷光/黄昏光）
- 提升图片质感（从手机照片到专业渲染图）
- 增加景深效果

#### 2.6 参考图融合
- 上传风格参考图 + 空间照片 → 融合生成
- 保持空间结构，套用参考图的风格和色调

**技术实现**：
- 图像生成：fal.ai / Replicate / Stability API
- Inpainting + ControlNet 保持空间结构
- 风格迁移：img2img + 风格提示词

---

### 📦 模块三：产品落地输出（Project Deliverable）

**目标**：让设计方案变成可执行的采购清单

#### 3.1 智能产品推荐
- 从效果图中的家具，匹配真实可购买产品
- 显示相似度评分
- 提供价格区间、购买链接
- 支持按预算筛选（经济型/中端/高端）

#### 3.2 产品清单（Bill of Materials）
- 自动生成表格：
  | 序号 | 品类 | 产品名称 | 规格尺寸 | 材质 | 参考价格 | 购买链接 |
- 支持导出 Excel / PDF
- 支持打印（A4排版）

#### 3.3 Moodboard 生成
- 自动提取方案中的色彩、材质、纹理
- 生成专业排版的情绪板
- 支持拖拽编辑、添加文字说明
- 导出高清图（用于客户汇报）

#### 3.4 方案对比
- 并排对比 2-4 套方案
- 高亮显示差异（换了哪些家具、色调变化）
- 客户投票/标记喜欢的方案

#### 3.5 预算估算
- 根据产品清单自动计算总价
- 按品类拆分（家具/灯具/布艺/饰品）
- 支持设置预算上限，自动推荐替代方案

**技术实现**：
- 产品匹配：向量检索（CLIP 特征）+ 电商API/自建产品库
- 清单导出：前端表格生成 + 后端 Excel 生成
- Moodboard：前端 Canvas/HTML 渲染

---

### 🛠 模块四：设计师工作流工具（Designer Toolkit）

#### 4.1 项目管理
- 按客户/项目组织方案
- 版本历史（修改记录、可回滚）
- 项目状态跟踪（构思中→方案确认→采购中→已交付）

#### 4.2 客户协作
- 生成客户分享链接
- 客户可评论、标记、投票
- 客户可一键导出喜欢的方案

#### 4.3 素材库
- 收藏常用家具单品
- 自建风格模板
- 常用色板保存

#### 4.4 灵感采集
- 浏览器插件：在 Pinterest/Houzz/小红书 上一键采集到素材库
- 自动分析采集图片的风格和色彩

## 六、与现有项目的差异对比

| 维度 | 当前版本（通用图片编辑） | 目标版本（软装 AI 工作台） |
|------|----------------------|------------------------|
| **用户** | 普通用户/泛设计师 | 软装设计师（专业用户） |
| **输入** | 任意图片 + 提示词 | 室内空间照片 |
| **核心能力** | 生成图片 | 理解空间 + 分析软装 + 设计方案 + 输出清单 |
| **交互** | 上传 → 写提示词 → 生成 | 上传 → AI 分析 → 点击操作 → 生成 |
| **输出** | 单张图片 | 方案图 + 产品清单 + Moodboard + 报价 |
| **工作流** | 单步操作 | 完整项目工作流（分析→设计→落地） |

## 十、总结：为什么要做这个方向？

1. **用户付费意愿强**：软装设计师是 B 端专业用户，愿意为效率工具付费
2. **工作流闭环**：从分析→设计→落地，覆盖完整工作流，用户粘性高
3. **技术可行性高**：多模态 LLM 的图像理解能力已经足够做空间分析
4. **市场空白**：国内没有专注软装设计的轻量级 AI 工具（酷家乐太重，美间不够智能）
5. **差异化壁垒**：积累的产品库、风格模板、用户数据会形成壁垒
